# BVA Analysis for i18n / Locale Support

## Specification

The Messages class provides localized strings for the application. Each supported locaale has its own set of tranlsations. The current locale defaults to English and can be changed at runtime through setLocale. Adding a new locale requires only a new translation file and no changes to existing code.

---

### Step 1 - Inputs and Outputs
- Input #1: target locale
- Output #1: current locale is updated

### Step 2 - Data Types
- Input #1: Reference
- Output #1: Reference
- Output #2: Case

### Step 3 - Concrete Values
- Input #1: English, Korean, null, unsupported locale
- Output #1: locale is set

- Test Case 2: setLocale rejects null
